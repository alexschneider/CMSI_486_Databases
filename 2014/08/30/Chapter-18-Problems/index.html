Chapter 18 Problems

Notes:

  • Index access helps to reduce the amount of time required to search through a file - stores each value and a pointer to all disk blocks that include it.
  • Primary index used to physically order files on the disk
  • Clustering index - used if multiple indexes are suitable for physical ordering
  • Secondary - doesn’t deal with ordering of the file, but locates it.
  • Dense indexes have an index entry for every single search key
  • Sparse - only some of the search keys
  • Indexes are useful because they occupy a smaller portion of the disk than the data file - they include fewer records and they only have two fields, the key and the address of the block.
  • Clustering - takes up more room, but works on more complex files and indexes
  • Secondary index - allows other means of accessing data when a primary index already exists - must still be unique.
    • Takes more storage space and a longer search time, but still improvement over no index at all.
    • For non-unqiue data, indexing is still possible, but makes searches more complicated, since the index has to store a list of pointers, instead of a single one.
  • Multilevel indexes
    • Used to reduce the search space for a binary search
    • Uses multiple levels of indexes until the highest level can fit in a single block.
    • Can be used for any index, so long as there are distinct values for K(i)
  • Partitioned hashing - n components produce n hashes, which are catted together to produce one hash. Useful for equality, but not really for searching
  • B-tree and B+-tree
    • Tree is formed of nodes, each node (except for the top node) has one parent and 0 or more children.
    • Trees are unbalanced
    • B-Trees
      • Kept between 50-100% full
      • Searching works by forcing things to the left of the key to be left than it, and things to the right of the key to be greater than it. That way, when you compare the key you are looking for to the tree, you can easily find out which direction to travel.
      • Balancing a tree minimizes the number of levels
      • B-trees have additional constraints to make sure that they are always balanced
      • Each node of a b-tree has multiple bits of data and multiple tree pointers - the heirarchy of increasing value goes left to right - tree pointer, data pointer, tree pointer, data pointer, etc, etc, and finally a tree pointer.
      • If removal causes the node to be less than half full, it’s combined with neighboring nodes
    • B+-trees
      • Like a b-tree, except each node only points to the node to the left most node at the next level, and the left node points to the node to the right, and so on
      • Insertions and deletions call for complicated algorithms, but they are computationally fast.
  • Grid files - multi-dimensional array. Works well for access time, but requires a lot of space and difficult to re-organize
  • Hash index - uses hashing other than the one used for primary index.
  • Bitmap indexes - index for one or more columns, and each value in the column is indexed.
    • Built using one particular value and is an array of bits (sequential 1s and 0s referenceing whether or not something exists at that location).
    • Efficient in storage space, but difficult to maintain
    • Can use bitwise operations to maintain though.
  • Function based indexing - applying a function on field(s) becomes the key to an index.
  • Logical vs Physical indexes
    • Physical indexes have a block location and an offset that must be changed if the file location on the disk is changed
    • Logical indexes - let some other system handle where the files are, and use indexes to point to it.
