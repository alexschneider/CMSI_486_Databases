Data Minining

One of the things I’ll be doing is adapting all of the Guide to Data Mining databases to use my database through pyORMatic. You can find the code in one of my repositories to be posted here.

  • Data mining
  • Our world is overloaded with data - previously we relied on experts to find data, but now, we have access to all the information ourselves.
  • Companies and groups use data about us to target narrow audiences directly, rather than appealing to a broad audience but losing some of the fringes.
  • Recommendation systems
    • Used by companies like amazon
    • Finding similar people based on previous interactions:
      • Compute distance on a grid (number of vertical cells and number of horizontal cells)
      • Similar people get similar recommendations
      • Finding the distance as the crow flies
    • Part of the issue with recommendation systems is trying to figure out how people weight their reviews - some people may only give as low as a 4, while some people may give reviews like 1s or 2s.
    • For large amounts of data (like iTunes music recommendation), the possibility of a matching scheme is slim, so when comparing things like tracks played, comparing “0”s is pointless
    • Cosine similarity gets rid of the 0 matches
    • Dense data should use manhattan and euclidean matching algorithms, but non-dense data with these algorithms provides faulty results. Sparse data works better with cosine similarity
    • Using k-nearest neighbor is another matching scheme
      • Compute “k” values using a prior mentioned algorithm, and provide them weight based on the strength of their score.

What I did with the program - adapt it to use pyORMatic rather than CSV/internal memory. I worked out most of the bugs with pyORMatic, but due to lack of time, I only was able to add flat file databases to pyORMatic. Hopefully that should come later.
